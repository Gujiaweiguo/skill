## 页面类型与布局模板

### Cover（封面）
| 元素 | 位置 | 说明 |
|------|------|------|
| 大标题 | 居中偏上 | 主题标题，最大字号 |
| 副标题 | 标题下方 | 补充说明或 slogan |
| 日期 / 场合 | 底部 | 小字，与底部保持边距 |
| 品牌元素 | 可选的 logo 或装饰 | 保持简洁 |

### TOC（目录）
| 元素 | 位置 | 说明 |
|------|------|------|
| "目录" 标题 | 左上或居中 | 明确标识 |
| 章节列表 | 主区域 | 3-6 项，带编号 |
| 装饰线/图形 | 章节间 | 视觉分隔 |

### Section（章节过渡页）
| 元素 | 位置 | 说明 |
|------|------|------|
| 章节编号 | 顶部偏左 | 大字数字 |
| 章节标题 | 章节编号右侧或下方 | 清晰标识 |
| 引用/数据 | 底部或侧面 | 一句引人思考的话或数据 |

### Content（内容页）
布局须主动切换，避免连续重复：

| 布局类型 | 适用场景 | 结构 |
|---------|---------|------|
| 左右分栏 | 对比、因果 | 左文右图 / 左图右文 |
| 大数字突出 | 关键数据 | 巨大数字 + 简短说明 |
| 时间轴 | 历程、发展 | 水平 / 垂直线性排列 |
| 对比图 | 前后、竞品 | 并列两栏对比 |
| 引用卡片 | 金句、用户证言 | 引号 + 加粗文字 + 来源 |
| 图标网格 | 特征列举 | 2×3 / 3×3 图标阵列 |
| **核心闭环** | **业务场景、流程闭环** | **详见下方"核心闭环版式"** |

### 核心闭环版式（Core Loop Layout）

适用于展示业务场景的完整流程闭环（如：停车联动、平台券营销、活动运营、积分经营）。

**页面结构（LAYOUT_WIDE 13.333×7.5）：**

```
┌──────────────────────────────────────────────────────────────────┐
│ [深色顶栏]  核心闭环 X/N  [场景标题]              [核心场景 徽章]  │
├──────────────────────────────────────────────────────────────────┤
│ [一句话副标题]                                                    │
│                                                                  │
│ ┌──────────────────────────────┐  ┌─────────────────────────┐   │
│ │ [流程图容器]                   │  │ [为什么要做]  [accent左栏] │   │
│ │  标题                          │  │  • 原因1                 │   │
│ │  ┌──┐→┌──┐→┌──┐→┌──┐→┌──┐→┌──┐│  │  • 原因2                 │   │
│ │  │01│ │02│ │03│ │04│ │05│ │06││  ├─────────────────────────┤   │
│ │  │图│ │图│ │图│ │图│ │图│ │图││  │ [怎么做]                  │   │
│ │  │标│ │标│ │标│ │标│ │标│ │标││  │  • 方法1                 │   │
│ │  │题│ │题│ │题│ │题│ │题│ │题││  │  • 方法2                 │   │
│ │  │描│ │描│ │描│ │描│ │描│ │描││  ├─────────────────────────┤   │
│ │  │述│ │述│ │述│ │述│ │述│ │述││  │ [有什么价值]              │   │
│ │  └──┘ └──┘ └──┘ └──┘ └──┘ └──┘│  │  • 价值1                 │   │
│ │  [标签1] [标签2] [标签3]        │  │  • 价值2                 │   │
│ └──────────────────────────────┘  └─────────────────────────┘   │
│ ┌────────────────────────────────────────────────────────────┐  │
│ │ [底部金句栏 — tone色填充]                                     │  │
│ └────────────────────────────────────────────────────────────┘  │
└──────────────────────────────────────────────────────────────────┘
```

**五大组件：**

| 组件 | 位置 | 尺寸 | 说明 |
|------|------|------|------|
| 顶栏（TopBand） | y:0, h:0.66 | 全宽 | 深色背景 + 标题（24pt加粗）+ 右上角徽章 |
| 流程图（FlowDiagram） | 左侧 60% | w:7.0, h:4.22 | 浅灰容器 + 6个步骤卡片 + chevron连接 + 底部标签 |
| 三卡解释（ThreeExplainCards） | 右侧 40% | w:4.62 | 竖排三张卡片：为什么要做/怎么做/有什么价值 |
| 底部金句（BottomStatement） | 底部 | 全宽 | tone色填充圆角条 + 居中加粗金句 |
| 副标题 | 顶栏下方 | w:11.0 | 12pt 灰色一行描述 |

**流程图步骤卡片规范：**
- 每个步骤：圆角矩形（w:0.96, h:2.08）+ 圆形图标 + 编号 + 标题 + 描述
- 步骤间用 chevron 箭头连接
- 底部最多3个 footerTag 标签（圆角药丸样式）

**配色系统（tone 切换）：**
每个核心闭环页使用独立的 tone 色，4个场景通常对应：
- 蓝色系（blue/blueSoft）— 如停车场景
- 珊瑚色系（coral/coralSoft）— 如营销/平台券场景
- 青色系（teal/tealSoft）— 如活动运营场景
- 琥珀色系（amber/amberSoft）— 如积分场景

**使用场景判断：**
当需要展示一个"从A到B的完整业务闭环"时，使用此版式。
不适合：纯数据展示（用大数字版式）、对比分析（用对比图）、简单列表（用图标网格）。

### Summary（总结页）
| 元素 | 位置 | 说明 |
|------|------|------|
| "总结" / "关键要点" | 顶部 | 标识 |
| 核心要点列表 | 主区域 | 3-5 个要点 |
| CTA / 联系方式 | 底部 | 行动号召 |

## 排版规范

### 字号参考
| 层级 | 字号(pt) | 加粗 | 颜色 |
|------|---------|------|------|
| 主标题 (Cover) | 36-48 | 是 | primary |
| 页面标题 | 24-32 | 是 | primary |
| 章节标题 | 20-24 | 是 | primary |
| 正文 | 14-18 | 否 | secondary |
| 标注/来源 | 10-12 | 否 | accent |
| 页码 | 10-12 | 否 | accent |

### 间距参考
| 位置 | 建议值 |
|------|--------|
| 页面边距 | 0.5"-0.75" |
| 标题与正文间距 | 0.3"-0.5" |
| 元素间距 | 0.15"-0.25" |
| 相关元素间距 | 近 (0.1"-0.15") |
| 无关元素间距 | 远 (0.3"+)

### 对齐规则
- 同一页面内统一使用左对齐或居中对齐
- 不要混用对齐方式
- 列表/要点保持相同缩进

## 六个设计原则

1. **一页一观点** — 每页只传达一个核心信息
2. **留白即呼吸** — 最少 30% 面积留白
3. **层次即导航** — 用字号、颜色建立信息层级
4. **对比即注意力** — 相邻元素字号差 ≥20%，标题为正文的 2-3 倍
5. **一致即专业** — 配色、字体、间距、风格统一
6. **多样即节奏** — 连续页不重复相同布局
