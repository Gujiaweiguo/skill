# PPT Master — Prompt 模板

## 修改场景 Prompt

```
用 slides 和 imagegen 编辑现有幻灯片。

输入 PPT 路径：{路径}
输出 PPT 路径：{路径}
logo 文件路径（如需要）：{路径}
需要修改的页码：{页码列表}
需要新增插图的页码：{页码列表}
品牌参考页：{路径（可选）}

先不要直接修改，先检查并输出：
1) 幻灯片尺寸与宽高比
2) 母版 / 版式结构
3) 主题字体、主题色、常用页边距与对齐规则
4) 每页主要元素位置与可编辑对象类型

确认后再执行修改：
1) 在每页右下角添加 logo，保持安全边距一致
2) 调整指定页面的布局
3) 在指定页面生成抽象插图
4) 按现有品牌风格新增幻灯片

完成后执行验证：
1) 渲染每页 PNG 审核
2) 检查元素是否越界或重叠
3) 检查字体缺失或替换
4) 输出修改后的 .pptx、生成脚本 .js、所用插图资产
```

## 新建场景 Prompt

```
用 slides 和 imagegen 创建新的演示文稿。

主题：{主题名称}
受众：{目标受众}
页数建议：{建议页数}
参考风格：{可选，品牌色/已有模板风格}
内容大纲：
{大纲内容}

先规划页面结构，输出：
1) 页面总数与每页类型（Cover / TOC / Section / Content / Summary）
2) 配色方案选择
3) 布局规划（确保连续页不重复）

确认后再生成：
1) 按规划逐页创建
2) 用 imageGen 生成封面视觉
3) 数据/图表保持为原生对象

完成后执行验证：
1) 渲染每页 PNG 审核
2) 检查布局是否按规划执行
3) 输出所有文件
```

## 文章转演示 Prompt

```
用 slides 和 imagegen 将以下文章转为演示文稿。

文章内容：
{文章全文或链接}

要求：
1) 提炼 3-8 个核心观点
2) 每个观点一页
3) 风格统一，适合{受众}

先输出提炼的观点列表和页面规划，确认后再生成。
```
