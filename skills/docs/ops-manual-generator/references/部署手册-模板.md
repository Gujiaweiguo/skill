### 2.2 安装 Docker 与 Compose

【标准内容】CentOS/Ubuntu 通用安装步骤：

```bash
# 卸载旧版本（如有）
sudo yum remove docker docker-client docker-client-latest docker-common latest -y 2>/dev/null || true
sudo apt-get remove docker docker-engine docker.io containerd runc 2>/dev/null || true

# CentOS 安装
sudo yum install -y yum-utils
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# Ubuntu 安装
sudo apt-get update
sudo apt-get install -y ca-certificates curl gnupg lsb-release
sudo mkdir -p /etc/apt/keyrings
curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

# 启动并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 验证
docker --version
docker compose version
```

VERIFY: `docker --version` 输出版本号 >= 20.10；`docker compose version` 输出 v2.x。

## 三、安装部署

### 3.1 获取部署包

【从 Dockerfile `FROM` + CI workflow 读取 + 需提供】说明镜像/代码获取方式：

**方式 A：使用预构建镜像**（推荐）

```bash
# 登录镜像仓库（如私有仓库）
docker login {registry-url} -u $REGISTRY_USER -p $REGISTRY_PASS

# 拉取镜像
docker pull {registry-url}/{系统名}:{version}
```

**方式 B：本地构建**

```bash
# 克隆代码
git clone {repo-url} /opt/{系统名}/src
cd /opt/{系统名}/src
git checkout v{version}

# 构建镜像
docker build -t {系统名}:{version} .
```

### 3.2 配置

【从 `.env.example` + docker-compose.yml `environment` 读取】配置步骤：

```bash

# 确认 .env 文件已正确填写（参考 1.5 环境变量表）
cat .env | grep -v '^#' | grep -v '^$'

# 确认 docker-compose.yml 中的挂载路径与实际目录匹配
# 【需提供】如有自定义配置文件（如 application.yml / nginx.conf），说明放置位置
```

VERIFY: `.env` 所有必填变量都有值（不含占位符 `<from .env>`）。

### 3.3 启动服务

【从 docker-compose.yml `depends_on` 读取启动顺序 + 标准命令】：

```bash
cd /opt/{系统名}

# 启动全部服务（compose 会按 depends_on 自动排序）
docker compose up -d

# 查看启动状态
docker compose ps

# 查看启动日志（如有异常）
docker compose logs -f --tail=100
```

VERIFY: `docker compose ps` 所有服务 STATUS 为 `Up` 或 `Up (healthy)`。

启动顺序说明（来自 compose depends_on）：
1. {数据库服务} —— 数据持久化层
2. {缓存服务} —— 依赖数据库
3. {后端服务} —— 依赖数据库 + 缓存
4. {Nginx} —— 反向代理，依赖后端

### 3.4 验证部署

【从 nginx.conf `location` + healthcheck 读取 + 需提供】：

```bash
# 健康检查端点
curl -sf http://localhost/health | jq .
# 或 HTTPS（部署 SSL 后）
curl -sfk https://localhost/health | jq .

# 关键业务接口冒烟测试
curl -sf https://{域名}/api/{关键接口} | jq .

# 检查 Nginx 状态
docker exec {nginx容器名} nginx -t
```

VERIFY:
- 健康检查返回 `{"status": "ok"}` 或 HTTP 200
- 关键业务接口返回有效数据
- `nginx -t` 输出 `syntax is ok` 和 `test is successful`

## 五、回滚

### 5.1 回滚条件

出现以下情况立即回滚：
- 升级后健康检查失败持续超过 5 分钟
- 核心业务接口报错率 > 5%
- 用户大面积反馈功能不可用

### 5.2 回滚步骤

```bash
cd /opt/{系统名}

# 1. 回退镜像版本
docker tag {registry-url}/{系统名}:{previous-version} {系统名}:latest
docker compose up -d --remove-orphans

# 2. 数据库回滚（仅当升级时有 schema 迁移）
# 【需提供】根据迁移工具给出回滚命令，例如：
# docker compose run --rm backend alembic downgrade -1
# 警告：数据库回滚可能导致新版本写入的数据丢失，务必先确认
```

VERIFY: 回滚后重复 3.4 验证部署，确认恢复到升级前状态。

## 六、附录

### 6.1 常见部署问题

【需提供 + 从历史 SOP 提取】列出部署时常见问题及解决方案。格式：

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 端口被占用 | 其他服务已占用 80/443 | `netstat -tlnp \| grep :80` 找到占用进程，停止或改端口 |
| 容器启动后立即退出 | .env 变量缺失或值错误 | 检查 `.env` 必填项；`docker logs {容器}` 看错误 |
| 数据库连接失败 | DB_HOST 配置错误或防火墙 | 确认 `.env` 的 DB_HOST 指向正确容器；检查容器网络 |
| Nginx 502 Bad Gateway | 后端服务未启动或端口不匹配 | `docker compose ps` 确认后端 Up；检查 nginx upstream 配置 |
| SSL 证书无效 | 证书过期或路径错误 | 更新证书；确认 nginx.conf 的 ssl_certificate 路径 |
