# PDF TOC Master — Prompt 模板

## 标准场景 Prompt

```
用 pdf-toc-master 给 PDF 生成目录书签。

输入 PDF 路径：{pdf_path}

先做 PDF 分析，输出：
1) 总页数、是否为扫描版
2) 是否有目录页（页码范围）
3) 印刷页码与 PDF 页码的偏移量
4) 现有书签情况

确认后再执行书签嵌入：
1) 从目录页提取标题与页码
2. 计算页码映射
3. 按章节分层生成书签
4. 嵌入到 PDF Outline

完成后执行验证：
1) 读取书签数量核对
2. 抽样 3-5 个条目确认跳转正确
3. 确认文件可正常打开
```

## 扫描版 PDF（需 OCR）

```
用 pdf-toc-master 给扫描版 PDF 生成目录书签。

输入 PDF 路径：{pdf_path}
输出 PDF 路径：{output_path}

这个 PDF 是扫描版（图片格式，无文字层）。
目录页在 PDF 第 {start} 到 {end} 页。

处理流程：
1. 先用 pdf2image 将目录页转为图片（300dpi）
2. 用 easyocr 识别中文标题和页码
3. 解析识别结果，提取「歌曲名 + 页码」对
4. 计算印刷页码到 PDF 页码的偏移量
5. 按章节分层生成书签并嵌入

输出前：
- 用文字列出所有提取到的目录条目，让我确认
- 标注 OCR 置信度较低的条目
```

## 文字版 PDF（有目录页）

```
用 pdf-toc-master 给 PDF 生成目录书签。

输入 PDF 路径：{pdf_path}

这个 PDF 是文字版，目录在 PDF 第 {start} 到 {end} 页。

处理流程：
1. 用 pypdf 提取目录页文本
2. 正则解析「标题 + 页码」
3. 计算页码偏移量（如有）
4. 分层生成书签并嵌入

完成后输出书签数量和抽样验证结果。
```

## 文字版 PDF（无目录页 — 标题推断模式）

```
用 pdf-toc-master 给这个 PDF 自动推断目录并生成书签。

输入 PDF 路径：{pdf_path}

这个 PDF 没有明确的目录页，需要从正文推断。

处理流程：
1. 遍历全文，按字号/字体聚类
2. 识别大字号单行文本作为可能的章节标题
3. 构建推断的 TOC 结构
4. 生成临时 TOC 让我确认
5. 确认后嵌入书签

注意：推断模式准确率取决于排版一致性。
输出时请标注推断置信度。
```

## 快速验证 Prompt

```
验证这个 PDF 的书签：
{pdf_path}

检查：
1) 书签总数
2) 书签层级结构
3) 抽样 3 个条目的跳转页码
4) 是否有损坏或异常
```
