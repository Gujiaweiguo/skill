# 果正商业会员营销系统采购服务 — 投标案例

当用户提到"果正投标""深圳果正""会员营销系统采购"或未另行指定路径时，参考此案例。

## 项目路径

```text
$BIDDING_DIR/果正商业会员营销系统采购服务/
├── content-packages/
│   ├── sz_gz_tender_技术标.docx                  # 招标文件原文
│   ├── sz_gz_tender_技术标.word-content.md        # 技术标内容包
│   └── sz_gz_tender_述标PPT.yaml                  # 述标PPT YAML（mode=tender）
└── sz_gz_tender_述标PPT_统一模板.pptx              # 述标PPT输出（17页）
```

## 招标文件来源

```text
$LANLNK_BASE/incoming/深圳果正会员招标文件/
├── 深圳市果正商业管理有限公司商业会员营销系统采购服务项目招标文件.docx
└── 深圳市果正商业管理有限公司商业会员营销系统功能清单.xlsx
```

## 述标PPT 结构（mode=tender，4段式）

| 章节 | 内容 |
|------|------|
| 封面 | 果正商业会员营销系统采购服务述标答辩 |
| 目录1 | 公司简介与资质（3页：公司概况、资质荣誉、团队架构） |
| 目录2 | 需求理解与方案蓝图（4页：需求理解、总体架构、功能覆盖、技术优势） |
| 目录3 | 重点功能响应（4页：会员管理、积分营销、数据洞察、AI赋能） |
| 目录4 | 实施服务与选择理由（4页：实施计划、服务保障、选择蓝联、感谢页） |

## 标准动作

1. 读取招标文件 + 功能清单，提取硬性要求、评分点、格式要求。
2. 输出技术标 Word 内容包 → word-master 排版生成技术标 .docx。
3. 输出述标PPT YAML（mode=tender）→ proposal-pptx compile.py 生成述标PPT。
4. 报价清单：openpyxl 按招标口径生成 .xlsx。
5. 输出响应矩阵与交付检查清单，标注需人工补充的事项。
