# Company Intro Generator — 公司介绍与供应商入围方案 Agent Pipeline

基于 `需求解析 + 标签匹配检索 + 智能编排 + 模板输出` 的方案，支持三种业务模式。

## 三种模式

| 模式 | 触发条件 | 输入 | 输出 | 使用阶段 |
|------|---------|------|------|---------|
| **A 初次介绍** | "生成公司介绍" / "做方案PPT" | 行业/业务线/场景描述 | 公司介绍PPT/Word | 客户初次接触 |
| **B 供应商入围** | "客户要方案报价" / "根据需求出方案" | 客户需求文档（Excel/Word） | 方案建议书PPT + 功能响应表 + 报价单 | 立项后、招标前 |
| **C 客户资产组合路由** | "按客户系统现状出介绍" / "4个版本" / "B路线" / "D路线" / "联系人转领导" | 客户已有系统资产（商管/会员 的有×无组合） | 4个版本公司介绍 Markdown/PPT（首访版） | 战略对齐面客，按客户资产选择首访主切口 |

## 配置读取

设置 `LANLNK_BASE` 环境变量指向素材根目录（默认当前目录），然后读取项目配置：
```yaml
# 路径: config/lanlnk.yaml（相对于 skill 根目录）
```

设置以下变量（后续所有路径引用均使用）：
- `$MATERIALS_DIR` = `$LANLNK_BASE/materials`
- `$PROPOSALS_DIR` = `$LANLNK_BASE/materials/14-proposals`
- `$BIDDING_DIR` = `$LANLNK_BASE/materials/15-bidding`

## 素材库结构

素材库位于 `$MATERIALS_DIR`，按类型分为 7 大类：

```
materials/
├── 01-company-overview/      公司概况（简介、历程、组织架构、业绩）
├── 02-qualifications/        资质证照（ISO、高新、软著、奖项）
├── 03-products/              产品方案（定位、功能、架构、优势）
├── 04-cases/                 客户案例（独立文件 + 带链接索引汇总）
├── 05-implementation/        实施方法论（实施流程、项目管理、培训）
├── 06-services/              服务体系（SLA、运维服务、数据安全）
├── 07-personnel/             项目人员（团队架构 + 个人档案）
└── raw/                      原始文档转换后的 Markdown（只读）
```

所有素材文件格式为 Markdown + YAML frontmatter，包含 id/type/domain/industry/scenario/tags 检索标签。

### 案例文件规范

案例已拆分为独立文件，每个案例包含 6 模块：
1. 项目背景 → 2. 需求分析 → 3. 实现功能 → 4. 实现效果 → 5. 项目图片 → 6. 合同信息

案例索引见 `$MATERIALS_DIR/04-cases/服务客户案例汇总.md`，按行业分类标注完整度。

### 人员文件规范

团队架构见 `$MATERIALS_DIR/07-personnel/项目团队架构.md`，
每个角色独立文件，含基本信息、职责、项目经验、证书、社保。部分人员姓名待补充。

### 5 种匹配级别

当素材满足以下条件时自动匹配：
- **精准匹配**：domain 标签完全匹配用户业务线 → 最高优先级
- **行业匹配**：industry 匹配 → 优先展示
- **场景匹配**：scenario/tags 匹配 → 补充展示
- **业务线扩展**：实施方法论/服务体系默认匹配所有业务线
- **兜底**：公司概况 + 资质 + 团队 → 必选

## Architecture

```
用户需求（自然语言描述）
        ↓
P0: 需求解析 + 环境检测

```

## P0: 需求解析与环境检测

### 0.1 模式判断

根据用户输入判断使用哪种模式：

| 信号 | 模式 |
|------|------|
| 提供了客户需求文档（Excel/Word/PDF） | **模式B 供应商入围** |
| "客户要方案报价" / "根据需求出方案" / "入围方案" | **模式B 供应商入围** |
| "按客户系统现状" / "4个版本" / "全家桶" / "B路线" / "D路线" / "客户有商管有会员" | **模式C 客户资产组合路由** |
| "生成公司介绍" / "做方案PPT" / 无客户需求文档 | **模式A 初次介绍** |

### 0.2 模式A：从需求提取参数

| 参数 | 说明 | 示例 |
|-----|------|------|
| 业务线 | 目标产品/方案领域 | 商管、会员、AI客服、AI问数、全栈 |
| 行业 | 目标客户行业 | 商业地产、零售、金融、政务 |
| 场景 | 业务场景 | 数字化转型、会员营销、私域运营 |
| 格式 | 输出格式 | Word、PPT |
| 篇幅 | 内容详细程度 | 简要版、标准版、详细版 |

### 0.3 模式B：解析客户需求文档

当用户提供需求文档（通常是 Excel）时，按以下步骤解析：

**步骤1：读取需求文档**
```bash
# Excel 格式 — 用 openpyxl 读取
python3 -c "
import openpyxl
wb = openpyxl.load_workbook('需求文档.xlsx', data_only=True)
for name in wb.sheetnames:
    ws = wb[name]
    for row in ws.iter_rows(values_only=True):
        print(' | '.join([str(v)[:80] if v else '' for v in row]))
"
```

**步骤2：提取需求条目**

将客户需求文档解析为标准化结构：
```
需求条目 = {
  序号: int,
  终端: "CRM后台 | 小程序 | APP",
  主模块: "会员管理 | 积分 | 营销 | 停车 | 活动 | ...",
  次模块: "具体功能名称",
  功能说明: "详细描述",
  匹配状态: "完全满足 | 部分满足 | 需定制 | 不支持"  ← P1阶段填充
}
```

**步骤3：提取关键参数**

从需求文档中推断：
| 参数 | 来源 |
|------|------|
| 客户名称 | 文件名或文档标题 |
| 业务线 | 根据需求内容判断（会员/商管/AI等） |
| 行业 | 根据客户背景判断 |
| 规模 | 根据需求中的商圈数、项目数判断 |
| 部署方式 | SAAS 或 私有化（根据需求或客户偏好） |
| 报价口径 | 首项目 + 新增项目数量 |

### 0.4 环境检测
```
1. 读取至少一份匹配素材 → 继续
2. 素材引用完整 → 内容包可生成
3. [模式B] 客户需求文档可读取 → 功能响应可生成
```

### 0.5 模式C：客户资产组合路由（对齐战略 v7）

**前置依赖**：先读取战略总报告路线定义
`$LANLNK_BASE/strategy/商业地产信息化战略/output/战略分析总报告.md` §1.2.1 两条路线。

**战略路线口径**（v7）：
- **B 路线（主战场）**：中腰部客户，已有商管/会员/POS/停车等系统 → "不换系统，用 AI 把已有系统用起来，降本、提效、增收"
- **D 路线（样板验证）**：低端/中小项目，无系统或系统弱 → "不是上一套传统系统，而是上 AI+商管/会员底座，让老板直接看到经营结果"

**4 版本路由表**（按客户"商管 / 会员"的有×无组合）：

| 版本 | 客户已有系统 | 战略路线 | 推荐产品组合 | 一句话卖点 |
|------|------------|---------|-------------|-----------|
| **V1** | 商管 ✓ + 会员 ✓ | B 路线 | 建议先验证：AI 激活已有商管与会员系统 | 不换系统，用 AI 把已有系统"用起来"，先选1-2个岗位场景验证 |
| **V2** | 商管 ✗ + 会员 ✓ | B 路线延伸 | 建议先验证：老板实时看到经营状况 | 先补齐关键经营数据，让老板能问数、看日报、追异常 |
| **V3** | 商管 ✓ + 会员 ✗ | B 路线延伸 | 建议先验证：会员资产沉淀与活动复盘闭环 | 参考正祥立项逻辑，先把停车、券、活动和会员身份打通 |
| **V4** | 商管 ✗ + 会员 ✗ | D 路线 | 建议先验证：轻量经营底座 + 一个 AI 经营样板 | 从一个经营问题开始，让老板先看到经营结果 |

**关键产品素材引用**：

| 产品 | 素材路径 | 用于版本 |
|------|---------|---------|
| 商业地产 AI Skills 增强服务（= LnkAgent 岗位 Skill 平台） | `$MATERIALS_DIR/03-products/商业地产AISkills增强服务.md` | V1-V4 全部 |
| 商管 ERP 系统（ShopEM） | `$MATERIALS_DIR/03-products/商管ERP系统.md` | V2、V4 |
| 商圈会员 CRM 系统 | `$MATERIALS_DIR/03-products/商圈会员CRM系统.md` | V3、V4 |
| 岗位 Skill 详细方案（营运/客服/招商/账龄/工单/企划） | `$CREAISKILL_BASE/04_岗位解决方案/*.md` | V1-V4 按 P0 主备选取 |

**AI 岗位 Skill 首访裁剪原则**：
- 4 个版本底层能力可以完整保留，但首访材料只呈现与客户现状最相关的 1-2 个验证场景。
- 联系人转领导场景下，PPT 必须让领导 3 分钟内看懂：不替换现有系统、解决岗位生产力问题、建议 30-60 天试点。
- 领导转发简版必须保留一页标杆客户 Logo 墙，建议标题为"蓝联服务过的商业地产与流通标杆客户"，放在"蓝联做什么"之后作为背书。
- 完整能力清单可放在后续方案或附录，不在首访前 5 页展开。

## P1: 素材检索与匹配

### 1.1 模式A：标签匹配

使用语义检索 + 标签匹配，按以下优先级筛选：

| 优先级 | 匹配条件 | 说明 |
|-------|---------|------|
| 1 | domain 标签匹配目标业务线 | 精准匹配用户要求的业务 |
| 2 | industry 标签匹配目标行业 | 同行业案例优先展示 |
| 3 | scenario / tags 匹配目标场景 | 匹配具体业务场景 |
| 4 | 兜底 | 所有通用素材（company-overview + qualifications）必选 |

### 1.2 模式B：功能响应匹配

将客户需求文档中的每条需求，与产品标准功能清单逐条匹配：

**产品功能清单来源**：报价单 Excel 中的"CRM模块清单"、"CRM标准功能清单"等 Sheet

**匹配规则**：
| 匹配状态 | 判定条件 | 标记 |
|---------|---------|------|
| ✅ 完全满足 | 产品标准功能已覆盖此需求 | 无偏离 |
| ⚡ 部分满足 | 产品覆盖核心能力，需少量配置 | 轻微偏离 |
| 🔧 需定制 | 产品未覆盖，需定制开发 | 需评估工期和费用 |
| ❌ 不支持 | 产品架构不支持此需求 | 明确偏离 |

**功能响应表输出格式**（Excel）：

| 序号 | 客户需求模块 | 客户需求功能 | 功能说明 | 我方响应 | 匹配状态 | 备注 |
|------|------------|------------|---------|---------|---------|------|
| 1 | 会员管理后台 | 顾客管理信息 | 支持查看客户基本信息... | CRM标准功能-会员列表已覆盖 | ✅完全满足 | |
| 2 | 积分策略 | 自定义抵现规则 | 设置积分价值、订单级抵扣 | CRM标准功能-积分规则已覆盖 | ✅完全满足 | |
| ... | ... | ... | ... | ... | ... | |

**匹配覆盖率统计**：完全满足 X% + 部分满足 Y% + 需定制 Z% = 总需求覆盖率

### 1.2 素材裁剪规则
- 简要版：保留公司概况 + 核心产品（最多3个） + 标杆案例（最多3个，优先选完整度 ✅ 的） + 资质摘要
- 标准版：保留匹配到的全部产品 + 匹配到的全部案例（优先 ✅ 再 ⚠️ 再 🔴） + 完整资质列表 + 实施与服务章节
- 详细版：保留所有匹配素材 + 保留所有产品细节 + 保留完整案例描述 + 团队介绍

### 1.3 案例智能选择
使用 case_matcher 自动匹配案例：
```bash
cd skills/business/material-importer && uv run scripts/case_matcher.py \
  --industry <客户行业> --scenarios <场景标签> --limit 5 --json
```
匹配维度：行业（×3）、场景（×2.5）、关键词（×1.5）、规模相近度（×1）、完整度（×0.5）。
也可用 `--list-tags` 查看所有可选行业和场景标签。

## P2: 内容编排逻辑

### 2.0 心理触发 + 咨询式写作原则

所有对外公司介绍先服务目标动作，而不是先介绍蓝联。生成前必须明确：读者看完后希望做什么。

| 目标动作 | 写作重点 |
|---|---|
| 经办人愿意转发 | 降低转发风险，给出转发理由和可复制话术 |
| 领导愿意见面 | 让领导看到经营价值、低风险试点和清晰下一步 |
| 客户愿意调研 | 让客户认同症状，愿意开放信息 |
| 客户愿意立项 | 证明问题值得解决，且可用小场景验证 |

领导转发简版使用固定说服链：

```text
目标动作 -> 读者心理 -> 趋势/危机 -> 症状假设 -> 未来收益 -> 蓝联解决路径 -> 蓝联可信证明 -> 低风险下一步
```

心理触发必须覆盖多类读者：

| 类型 | 心理驱动 | 材料触发方式 |
|---|---|---|
| 跟随型 | 怕落后、怕同行已做 | 行业趋势、同业动作、危机感 |
| 领先型 | 想创新、想做样板 | 政策号召、技术成熟、可见效果 |
| 风险规避型 | 怕项目失败、怕影响现有系统 | 只读接入、小范围试点、本地部署、人工兜底 |
| 结果导向型 | 只关心值不值得 | 经营指标、节省时间、异常发现、ROI验证 |

买药与保健品逻辑：

- 买药：先提出客户可能认同的症状，再给解决路径。例：领导问数慢、报表人工重、活动复盘难、客服咨询重复、数据分散。
- 保健品：激发更好状态的向往。例：领导随时看经营、已有系统价值放大、形成AI应用样板、岗位少做重复劳动。

咨询式页面规则：

- 标题必须是判断句，不写泛泛的“产品介绍”“公司简介”。
- 每页只传达一个核心观点。
- 先给结论，再给分解和证据。
- 用“我们对贵司这类客户的假设”表达症状，避免未经验证地诊断客户。
- 心理触发、买药、保健品、跟随型、领先型、经办人转领导等词只用于内部创作判断，严禁出现在对外PPT正文和文件名里。

蓝联可信证明必须形成四段链条：

| 证明类型 | 推荐页面 |
|---|---|
| 行业证明 | 标杆客户Logo墙 |
| 场景证明 | 岗位Skill全景和重点场景 |
| 系统证明 | 底层已有系统 + 中层岗位Skill + 上层运营服务三层图 |
| 交付证明 | 30-60天试点路径和风险控制 |

广州海港城这类“已有商管 + 会员系统，联系人转领导”场景，PPT建议结构：

```text
P1 封面：不换系统，用AI让已有商管与会员系统产生经营结果
P2 为什么值得转给领导
P3 行业趋势与危机感
P4 我们对已有商管+会员客户的症状假设
```

   ```bash
   cd $SKILL_DIR/skills/ppt/ppt-master/templates/proposal-pptx
   uv run compile.py "$PROPOSALS_DIR/content-packages/{项目名称}_intro-pptx.yaml"
   ```
4. 输出到 `$PROPOSALS_DIR/{项目名称}/蓝联科技公司介绍_{客户简称}_{YYYYMMDD}.pptx`

**YAML 内容包结构（intro 模式）：**

```yaml
mode: intro
base_ppt: "/opt/code/docs/lanlnk/incoming/正祥选型方案/蓝联科技CRM商圈会员数智营销方案_202604.pptx"
output: "$PROPOSALS_DIR/{项目名称}/蓝联科技公司介绍_{客户简称}_{YYYYMMDD}.pptx"

cover:
  subtitle: "{客户全称}"
  date: "{YYYY年M月}"

toc:
  titles:           # 目录页标题（4项以内）
    - 关于蓝联
    - "{定制章节1}"
    - "{定制章节2}"
    - 合作展望

theme:
  primary: "#0A1F3D"
  accent: "#2196F3"
  font: "Microsoft YaHei"

slides:              # 客户定制页（插在公司介绍+案例之后）
  - type: text-bullets
    title: "{标题}"
    body: "{正文}"
    items: ["要点1", "要点2"]
  - type: feature-cards
    title: "{标题}"
    columns: 3
    items:
      - title: "{卡片标题}"
        body: "{卡片描述}"
```

**可用 slide 类型：**
- `text-bullets` — 标题+正文+要点列表
- `feature-cards` — 标题+卡片网格（columns: 2/3/4）

**输出结构（intro 模式）：**
- P01 封面（参考 PPT cover，母版=标题幻灯片）
- P02 目录（参考 PPT TOC，母版=节标题，文本已修改）
- P03-P20 公司介绍+案例（参考 PPT section1 的18页，含真实产品截图/案例图）
- P21-PN 客户定制页（YAML slides，母版=标题和内容）
- P(N+1) 为什么选择蓝联（参考 PPT idx=35）
- P(N+2) 感谢观看（参考 PPT idx=61，母版=标题幻灯片）

### 3.3 Word 生成：输出结构化 Word 内容包

**不再使用 python-docx 自行生成 Word。** 改为按 [Word 内容包规范](../../word/word-master/reference/word-content-package-spec.md) 输出 `.word-content.md`，交给 word-master 做专业排版。

**流程：**
1. P2 内容编排完成后，按 Word 内容包规范生成 `.word-content.md`
2. 放入 `$PROPOSALS_DIR/content-packages/{项目名称}_产品方案书_{YYYYMMDD}.word-content.md`
3. 告知用户："内容已编排完成，正在调用 word-master 排版生成产品方案书..."
4. word-master 读取内容包后自动生成 `.docx`，输出到 `$PROPOSALS_DIR/{项目名称}/`

**公司介绍/方案书章节 → 内容包映射：**

| 文档章节 | 内容包元素 | 说明 |
|---------|-----------|------|
| 封面 | `cover:` 元数据 + `template: proposal` | 公司介绍封面 |
| 目录 | 自动生成 | word-master 根据 heading-1 生成 |
| 公司简介 | `heading-1` + 正文 + 配图引用 | 分页 |
| 行业洞察 | `heading-1` + bullet list | 分页 |
| 产品方案 | `heading-1` + `heading-2` 每产品一节 | 分页，可选 `table: function-matrix` |
| 实施方法 | `heading-1` + 流程描述 | 分页 |
| 服务体系 | `heading-1` + 服务项目列表 | 分页 |
| 客户案例 | `heading-1` + 每个案例一页 | 分页，含表格 |
| 资质荣誉 | `heading-1` + 资质列表 | 可合并到公司简介 |
| 团队介绍 | `heading-1` + `table: personnel-matrix` | 可选 |
| 联系方式 | `heading-1` | 结尾页，不必须分页 |

## 模式B 完整工作流（供应商入围）

```
客户需求文档（Excel/Word）
        ↓
P0: 需求解析（openpyxl 读取 → 提取需求条目 → 推断业务参数）
        ↓
P1: 功能响应匹配（需求 × 产品功能清单 → 逐条匹配 → 功能响应表）
        ↓
P2: 内容编排（需求理解 → 方案概述 → 功能概览 → 架构 → 报价 → 服务 → 案例）
        ↓
P3: 文档生成
    ├── PPT：输出 YAML 内容包 → proposal-pptx compile.py（mode=intro）→ 方案建议书.pptx
    ├── 功能响应表：openpyxl 生成 .xlsx
    └── 报价单：openpyxl 按标准4分类格式生成 .xlsx
        ↓
P4: 输出核验（覆盖率检查 + 金额校验 + 完整性检查）
```
