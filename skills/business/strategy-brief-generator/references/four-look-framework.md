# 四看战略分析框架

本文件是 strategy-brief-generator 的核心方法论参考。明源战略模板归纳后，统一落到这里作为分析框架的权威定义。

## 框架来源

明源战略模板（多份资料归纳）。明源在两个角色中出现：

1. **方法论模板**（`input/00-methodology/明源战略模板/`）→ 归纳到本文件
2. **商管竞品**（`input/03-competitors/商管系统/明源/`）→ 作为竞品证据

本文件只承载第一个角色。

## 四看 → 定战略 → 执行计划

### 一、看市场（Market）

回答：**市场正在发生什么变化？客户面临什么压力？**

| 分析维度 | 关注问题 | 资料来源 |
|---------|---------|---------|
| 行业趋势与宏观环境 | 存量/增量、消费降级/升级、数字化渗透率 | `input/01-market/商业地产市场趋势/` + web search |
| 客户变化与需求演变 | 商管公司从"收租"到"经营"、会员运营刚需化 | `input/01-market/` + SOP |
| 技术变化 | AI/RAG/BI/低代码/私有化大模型趋势 | `input/01-market/AI趋势/` + web search |
| 政策与经营压力 | 降本增效、合规、数据安全、国资监管 | `input/01-market/` + web search |

**输出要求**：每条市场信号标注来源；区分"行业共识"和"少数观点"。

### 二、看竞对（Competitors）

回答：**竞品在做什么？他们的打法和弱点是什么？**

商管和会员分开分析，因为竞争逻辑不同。

#### 商管竞品（明源/海鼎/安盛旗/酆泽/科传）

| 分析维度 | 关注问题 |
|---------|---------|
| 竞品定位与产品矩阵 | 全套 ERP？垂直商管？SaaS 还是私有化？ |
| 核心能力与特色打法 | 招商/合同/财务/物业哪个模块最强？ |
| 客户打法与生态策略 | 主攻头部集团？中部？代理渠道？ |
| AI/数据能力 | 有没有 AI问数/BI/智能分析？ |
| 价格/交付/服务模式 | 标准产品？定制？实施周期？ |

#### 会员竞品（猫酷/灵智数科/海鼎）

| 分析维度 | 关注问题 |
|---------|---------|
| 会员体系设计 | 等级/积分/权益模型 |
| 营销与私域能力 | SCRM/社群/小程序/商城 |
| 数据与 CDP | 会员画像、标签、分群、自动化 |
| AI/智能营销 | 智能推荐、AI导购、智能客服 |

**输出要求**：竞品能力矩阵 + 每家竞品的战略主张 JSON。

### 三、看自己（Self）

回答：**蓝联现在有什么？能做什么？缺什么？**

| 资产 | 盘点维度 |
|------|---------|
| 商管（源码 + PRD） | 已有模块、能力成熟度、技术栈、与竞品差距 |
| 会员（materials 资料） | 产品功能、已交付案例、行业覆盖、与竞品差距 |
| AI（lnkchatbi / langchat 源码） | AI问数能力、RAG 能力、大模型集成、与竞品差距 |
| 案例与交付 | 标杆案例、实施方法论、服务团队、客户口碑 |
| 组织与资源 | 团队规模、技术能力、资金、合作伙伴 |

**关键原则**：诚实面对短板。不夸大已有能力，不回避明显缺口。

### 四、看机会（Opportunity）

回答：**市场缺口在哪？蓝联的优势交叉点在哪？AI 新机会在哪？**

| 机会类型 | 推导逻辑 |
|---------|---------|
| 市场缺口 | 客户需要但没人做好的（如：商管+会员一体化、中小商管数字化） |
| 竞品弱点 | 竞品做不好或没做的（如：AI问数、RAG、移动端体验） |
| 自身优势交叉点 | 蓝联商管 + 会员 + AI 三块资产的组合优势 |
| AI/RAG 新机会 | 从 SOP 推导出的 AI问数场景 + RAG 知识场景 |

**输出要求**：每个机会点标注市场规模感知（大/中/小）、竞争激烈度、蓝联匹配度。

### 五、定战略（Strategy）

基于四看，输出战略定位。

| 战略要素 | 内容 |
|---------|------|
| 战略定位声明 | 一句话讲清蓝联在商业地产信息化领域的位置 |
| 目标客户画像 | 主攻哪类客户（头部集团/中部商管/中小商管） |
| 差异化主张 | 相对明源/海鼎/猫酷，蓝联的独特价值 |
| 产品组合策略 | 商管 + 会员 + AI 怎么打包、怎么卖 |

**关键原则**：战略定位要能被一句话讲清。讲不清说明没想清楚。

### 六、执行计划（Execution）

把战略变成可执行的动作。

| 动作类型 | 内容 |
|---------|------|
| 阶段目标 | P0（最小闭环）/ P1（增强）/ P2（深度） |
| 产品路线图 | 先做什么模块、后做什么模块 |
| 市场与售前动作 | 标杆案例打造、行业活动、售前物料 |
| 交付能力建设 | 实施团队、合作伙伴、SaaS 化 |

## 框架使用注意

1. **四看不是线性流程**：分析过程中可能反复回到前面的"看"补充证据。
2. **看自己要诚实**：高估自己能力是战略分析最常见的错误。
3. **看机会要具体**：不要写"AI 是大机会"这种空话，要写"商管经营分析的 AI问数，因为 SOP 暴露了 X 个管理动作需要即时数据查询"。
4. **定战略要收敛**：四看分析可能发现 10 个机会，战略只能选 2-3 个重点。
5. **执行计划要可验证**：每个阶段目标要有验收标准。
