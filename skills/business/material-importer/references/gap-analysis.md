# 资料缺口审查参考

此文件定义 P6 双模式资料缺口审查的完整框架，由 material-importer 在需要时加载。
仅在用户触发缺口审查需求时加载此文件。

---

## 双模式说明

| 模式 | 目标 | 典型问题 | 输出产物 |
|------|------|---------|---------|
| **方案介绍模式** | 能否做出让客户买单的提案 | "面向商业地产做会员方案还缺什么" | 素材覆盖率 + 内容质量报告 |
| **投标模式** | 能否拿下这个标 | "XX项目投标还缺什么材料" | 符合性检查 + 缺口清单 + 行动建议 |

## 方案介绍资料缺口审查

### 需求场景拆解

| 参数 | 说明 | 示例 |
|------|------|------|
| 目标行业 | 客户所在行业 | 商业地产、零售、产业园 |
| 目标业务线 | 要介绍的产品/方案 | 商管、会员、AI客服、AI问数 |
| 目标场景 | 客户痛点场景 | 会员营销、私域运营、数字化转型 |
| 客户画像 | 客户规模/类型 | 大型购物中心、连锁品牌 |
| 输出格式 | 最终要生成的格式 | PPT/Word |

### 5 层覆盖率评分

```
┌─────────────────────────────────────────────────────┐
│  方案介绍 5 层覆盖模型                              │
├─────────────────────────────────────────────────────┤
│                                                       │
│  第 1 层：吸引注意（30 秒电梯演讲）                   │
│    - 公司一句话定位 + 差异化优势                      │
│    - 行业理解 + 客户痛点洞察                          │
│    - 衡量标准：公司简介 + 产品定位描述                     │
│                                                       │
│  第 2 层：建立信任（1 分钟）                          │
│    - 资质背书（高企、双软、ISO、软著）                │
│    - 服务规模（客户总数、面积覆盖、续约率）           │
│    - 商业生态（腾讯/阿里/微信支付等合作伙伴）         │
│    - 衡量标准：资质证照 + 客户覆盖 + 生态伙伴            │
│                                                       │
│  第 3 层：方案匹配（2 分钟）                          │
│    - 目标客户痛点的产品方案                            │
│    - 功能清单 + 技术架构 + 差异化优势                  │
│    - 实施方法论 + 服务体系保障                         │
│    - 衡量标准：产品方案 + 实施方法论 + 服务体系           │
│                                                       │
│  第 4 层：效果证明（2 分钟）                          │
│    - 同行业/同场景标杆案例                            │
│    - 量化数据（会员增长、交易提升、效率提高）         │
│    - 客户证言 / 感谢信 / 续约证据                     │
│    - 衡量标准：匹配案例 + 效果数据                        │
│                                                       │
│  第 5 层：团队支撑（30 秒）                           │
│    - 项目团队资历 + 人员证书                          │
│    - 服务承诺 + SLA                                   │
│    - 衡量标准：人员档案 + 服务承诺                        │
│                                                       │
└─────────────────────────────────────────────────────┘
```

| 🟡 部分 | 有素材但缺细节/数据 | |

## 投标资料缺口审查

### 投标检查维度

| 维度 | 素材来源 | 典型投标要求 |
|------|---------|------------|
| 企业资质 | 02-qualifications/ | 高新、双软、ISO、等保、系统集成资质 |
| 服务业绩 | 01-company-overview/近三年服务业绩 + 04-cases/ | 同类项目合同、面积/金额达标 |
| 人员配置 | 07-personnel/ | 项目经理 PMP、技术负责人高级职称、社保连续 X 个月 |
| 财务状况 | 01-company-overview/（财报） | 营收规模、资产负债率、审计报告 |
| 体系认证 | 02-qualifications/ | ISO9001/ISO27001/ISO20000 等 |
| 知识产权 | 02-qualifications/ | 软著数量、专利数量 |
| 安全资质 | 02-qualifications/ | 等保测评、信息安全认证、数据安全能力 |
| 服务承诺 | 06-services/ | SLA 响应时间、驻场服务、质保期 |
| 实施保障 | 05-implementation/ | 项目计划、质量管理、风险管控 |

### 符合性评分标准

| 评分 | 含义 | 对投标的影响 |
|------|------|------------|
| 🟢 满足 | 有合格素材且在有效期内 | 不影响 |
| 🟡 部分满足 | 有但有效期临近/内容不完整 | 可能扣分，需留意 |
| 🔴 不满足 | 无相关素材或已过期 | 可能废标 |
| ⚫ 未知 | 用户未提供招标要求，无法判断 | 待确认 |

### 投标缺口报告模板

```
═══════════════════════════════════════════════════════
  投标资料缺口审查报告
═══════════════════════════════════════════════════════
  目标项目：（需用户提供招标文件或要求）
  审查日期：2026-06-15
  数据来源：素材库现有内容
═══════════════════════════════════════════════════════

一、企业资质 ─── 🟡 部分满足
  ✅ 国家高新技术企业（有效至 2026-08-30，剩余 75 天 ⚠️）
  ✅ 双软认证（有效期未知）
  ✅ ISO体系认证（有效至 2027-10-15）
  ⚠️ 缺等保测评/信息安全等级保护 → 部分项目必备
  ⚠️ 缺软件著作权清单（已有 30+ 项待录入）→ 建议批量导入

二、服务业绩 ─── 🟢 满足
  ✅ 近三年 8 个项目，覆盖 2.7-136 万㎡
  ✅ 国企客户（广州城投、粤海天河城）、上市客户
  ✅ 合同信息基本完整

三、人员配置 ─── 🔴 不满足
  ❌ 7 个核心岗位均缺具体姓名
  ❌ 无证书扫描件（PMP、软考等）
  ❌ 无社保证明
  → 无法通过人员资格审查，此为当前最大风险点

四、财务状况 ─── 🔴 不满足
  ❌ 无财务报表、审计报告、营收数据
  → 部分招标项目要求近三年审计报告，需财务部门准备

五、体系认证 ─── 🟡 部分满足
  ✅ ISO 系列认证有
  ⚠️ 缺 ISO20000（信息技术服务管理体系）→ 部分 IT 项目要求
  ⚠️ 缺 CMMI 认证（部分大型国企要求）

六、知识产权 ─── 🔴 不满足
  ❌ 30+ 项软件著作权未录入素材库
  ❌ 无专利信息
  → 软著是高新企业支撑材料，应尽快批量入库

七、安全资质 ─── 🔴 不满足
  ❌ 无等保测评报告
  ❌ 无数据安全能力认证
  → 政务/国企项目基本要求等保二级以上

八、服务承诺 ─── 🟢 满足
  ✅ SLA 分级（一级 30 分钟响应 / 二级 2 小时 / 三级 4 小时）
  ✅ 7×24 小时服务
  ✅ 服务体系完整

九、实施保障 ─── 🟡 部分满足
  ✅ 实施方法论完整（8 层体系）
  ⚠️ 缺开发流程与工具细节（CI/CD、测试体系、技术栈）
  ⚠️ 缺风险管控具体预案

═══════════════════════════════════════════════════════
📋 投标风险评分
═══════════════════════════════════════════════════════
  综合符合率：约 45%
  可投标类型：中小型商业项目（会员系统、小程序开发）
  不建议投标：大型政企 IT 项目（资质和人员缺口大）

  🔴 致命缺口（导致废标）：
    1. 人员姓名 + 证书 + 社保 → 立即向 HR 收集
    2. 财务报表/审计报告 → 向财务部门索取

  🟡 扣分缺口（影响评分）：
    1. 软件著作权清单（30+ 项未录入）
    2. 等保测评/安全资质
    3. 高企有效期仅剩 75 天（需准备续期）
    4. 案例效果数据不全

  🟢 已有优势：
    1. SLA 服务体系完整
    2. 实施方法论体系完整
    3. 近三年业绩有 8 个项目可展示

═══════════════════════════════════════════════════════
🎯 建议立即行动
═══════════════════════════════════════════════════════
  1. HR/行政：收集 7 个核心岗位的姓名、简历、证书、社保
  2. 行政：批量导入 30+ 项软件著作权（已有清单）
  3. 财务：准备近三年审计报告/财报
  4. 技术：补充开发流程、风险管控细节
  5. 运维：高企证书续期处理
```

## 双模式对比速查

| 对比项 | 方案介绍模式 | 投标模式 |
|--------|------------|---------|
| 关注核心 | 故事性 + 说服力 | 符合性 + 合规性 |
| 输出物 | 5 层覆盖率 + 内容质量 | 9 维度符合性 + 风险评分 |
| 典型评分 | 🟢充足/🟡部分/🔴缺失 | 🟢满足/🟡部分/🔴不满足/⚫未知 |
| 行动建议风格 | "建议补充XX让方案更有说服力" | "XX缺口可能废标，立即处理" |
| 适用阶段 | 售前方案制作 | 投标前准备 |
| 触发方式 | "检查素材够不够做XX方案" | "检查XX项目投标还缺什么" |
