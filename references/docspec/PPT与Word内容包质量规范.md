# PPT 与 Word 内容包质量规范

## 目标

内容包是上游业务 skill 与渲染 skill 之间的契约。质量重点是结构正确、素材可用、渲染可检查，而不是只看正文是否顺眼。

## Word 内容包

必须保证：

- YAML frontmatter 可解析。
- `title`、`type`、`template` 等关键字段完整。
- 章节层级连续，表格类型与数据匹配。
- 图片和素材路径存在，缺失时显式标记。
- 表格列数一致，金额、日期、项目名全文一致。
- 渲染失败时仍交付可读 Markdown 内容包和手工套版说明。

## PPT 内容包

必须保证：

- 每页只有一个核心观点。
- 章节顺序符合目标场景：汇报、方案、投标、培训、技术分享。
- 数据页有来源；示意图页不伪装成真实系统截图。
- YAML / Markdown 表格不用会破坏解析的特殊列名。
- 视觉素材路径存在；没有素材时用结构化占位，不编造截图。

## 正式渲染验收

| 类型 | 验收方式 |
|---|---|
| Word | 内容包校验器、目录/页码/表格/图片人工抽查 |
| PPT | 编译生成、打开检查封面/目录/关键页/图片/溢出 |
| Excel | 公式、合计、税率、优惠空位、sheet 命名检查 |

## 红线

- 不把渲染器错误归咎为内容已完成。
- 不交付无法打开的 Office 文件作为最终结果。
- 不在内容包里硬编码一次性绝对路径，除非该路径是明确的素材库权威路径。
- 不用截图替代本应可编辑的文字、表格和数据。
