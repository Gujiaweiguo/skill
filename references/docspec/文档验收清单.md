# 文档验收清单

交付前按本清单检查。未通过项必须进入待确认、缺口或风险说明。

## 通用检查

| 检查项 | 通过标准 |
|---|---|
| Doc Brief | 文档类型、读者、用途、来源、边界、验收方式明确 |
| 来源 | 关键事实有来源或明确标待确认 |
| 结构 | 标题层级清楚，同级章节粒度一致 |
| 术语 | 同一概念名称一致，有必要时提供术语表 |
| 覆盖 | 输入需求、评分点、能力项或操作路径有映射 |
| 风险 | 缺资料、冲突、假设、待确认项未被隐藏 |
| 下游 | 使用方式、输出路径、下一步动作明确 |
| 敏感信息 | 无密码、token、连接串、个人敏感信息明文 |

## 类型检查

| 类型 | 额外检查 |
|---|---|
| PRD | 状态、置信度、证据表、差距分析、review 清单齐全 |
| 方案 | 客户画像、产品边界、案例证明等级、素材来源清楚 |
| 投标 | 响应矩阵、废标风险、资格缺口、报价一致性已检查 |
| 战略 | 事实/判断/建议分层，假设明确 |
| 手册 | 步骤可执行，截图/界面/文本一致，知识库版规则已处理 |
| Word/PPT | 内容包可解析，素材存在，渲染后抽查关键页 |
| Skill | 触发边界、输入输出、流程、失败模式、维护规则齐全 |
