# 文档类 Skill 适配规范

## 适配目标

文档类 skill 必须把 DocSpec 变成执行流程的一部分，而不是在结尾附一个“检查质量”的口号。

## 接入点

每个文档类 skill 至少在四处接入 DocSpec：

| 位置 | 要求 |
|---|---|
| 输入解析 | 建立 Doc Brief，识别文档类型、读者、来源、边界 |
| 生成流程 | 产生证据表、响应矩阵、素材映射或内容包 |
| 质量校验 | 按 `文档验收清单.md` 做交付前检查 |
| 复利回流 | 发现跨 skill 规则时触发 compound-learning 写回 DocSpec |

## 各类 skill 适配重点

| skill 类型 | 必须额外保证 |
|---|---|
| PRD | 需求、代码、竞品、客户资料分源；状态和置信度一致；待确认项不并入最终结论 |
| 公司介绍/方案 | 客户画像、素材匹配、案例边界、证明等级清楚 |
| 立项报告 | 决策问题、投入产出、风险、替代方案和推荐方案分开 |
| 投标 | 招标要求逐条映射，废标风险优先，未知信息不得补写 |
| 战略简报 | 事实、判断、建议分层，市场/竞品/自身能力证据分源 |
| 需求评估 | 覆盖率、二开范围、成本基准和报价基准分开 |
| 操作手册 | 运行表面验证优先，截图/步骤/知识库版一致 |
| 部署维护手册 | IaC 事实、人工经验、敏感信息脱敏、回滚/故障步骤可执行 |
| Word/PPT | 内容包结构正确，素材引用存在，渲染后可检查 |
| 报价单 | 定价来源、优惠空位、税率、合计公式、第三方集成边界清楚 |

## Skill 内推荐写法

在 `SKILL.md` 的质量或验收章节加入：

```markdown
## DocSpec 质量基线

本 skill 生成的文档必须遵守 `/opt/code/skill/references/docspec/`：
- 写前建立 Doc Brief。
- 生成时保留来源、证据、待确认项和风险。
- 交付前按 `文档验收清单.md` 验收。
- 可复用质量经验通过 `compound-learning` 回流到 DocSpec 或本 skill references。
```

## 复利回流条件

下列情况必须考虑回流：

- 同一种质量问题在 2 个以上文档类 skill 出现。
- 新增了通用证据等级、状态词、验收口径或内容包约束。
- 某个客户/项目暴露出通用模板缺陷。
- 某个文档产物被下游使用失败，原因属于结构/证据/格式问题。
